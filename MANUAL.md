# 運用マニュアル (Operations Manual)

このサイトの更新手順、品質基準、トラブルシューティングをまとめた管理者用ドキュメントです。

## 1. 更新手順

管理画面 (`/admin`) から全てのコンテンツをブラウザのみで更新できます。
ログインURL: `/admin` (Googleアカウント認証)

### Works (実績)
- **追加**: タイトル、URL、技術スタック、サムネイル画像が必須です。
- **画像**: 推奨サイズ 1200x630 (OGP比率)。2MB以下。
- **公開**: 「Review / Public」スイッチで即時公開されます。

### Blog (記事)
- Markdown記法が使えます（見出し、リスト、リンク等）。
- 外部記事（Zenn/Qiita）ではなく、自分の言葉で書く場所として設計されています。

### Products & Devices
- **Products**: 販売物やテンプレート。価格表示は自由記述です。
- **Devices**: 使用機材。カテゴリは自由記述ですが「PC」「Audio」「Desk」などで統一すると綺麗です。

### Profile & Settings
- **Profile**: サイト全体の自己紹介文やアバターを変更できます。
- **Links**: X (Twitter), GitHub, YouTube等のリンクを追加・並び替えできます。

## 2. 定期バックアップ（超重要）

このサイトは Supabase (無料枠) を使用しています。
万が一のデータ消失や移行に備え、**月1回** または **大量更新後** には必ずバックアップを取ってください。

1. `/admin/settings` にアクセス
2. **"Export Data"** (Data Management) をクリック
3. ダウンロードされた `.json` ファイルをローカル（安全な場所）に保存

復元する際は、同画面の **"Import Data"** からJSONファイルをアップロードしてください（上書きされます）。

## 3. デザイン・品質基準

### モーション
- **Fast-first**: 演出のために操作を待たせない。
- 原則、すべてのボタンやリンクは `hover` 時に反応があるようにしてください。

### コンテンツ
- **Who -> Wait -> What**: 誰か(Home) → 興味(Works) → 詳細(About) の順で読ませる。
- 専門用語は使いすぎず、採用担当者（非エンジニア）にも伝わる平易な言葉で。

## 4. トラブルシューティング

### 画面が真っ白になる
- ブラウザのキャッシュをクリアしてください。
- どうしても直らない場合、開発者に連絡し Vercel のログを確認します。

### ログインできない
- `ADMIN_EMAIL` 環境変数で設定された Google アカウント以外ではログインできません。
- Vercel の Environment Variables 設定を確認してください。

### 画像がアップロードできない
- ファイルサイズが 2MB を超えていませんか？
- 形式は JPG, PNG, GIF, WebP のみ対応しています。
